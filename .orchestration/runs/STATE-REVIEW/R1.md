### CHOICES MADE (conventions, model decisions, defaults — with rationale + where set)
- Scale convention choice: core geometry treats `Lambda_IR` as `1/z_v`, while physical first gauge KK mass is `2.448687... * Lambda_IR`; quark bookkeeping default is `M_KK = xi_KK * Lambda_IR` with `DEFAULT_QUARK_XI_KK = 1.0`, but publication/full catalog scan defaults use `xi_KK = 2.4487` (`quarkConstraints/scales.py:20`, `quarkConstraints/scales.py:21`, `quarkConstraints/scales.py:33`, `quarkConstraints/scales.py:49`, `scripts/run_full_catalog_scan.py:145`, `scripts/run_rs_anarchy.py:85`). Rationale: keep old quark helper bookkeeping explicit while allowing physical first-KK convention in scans. Alternative: force one convention globally; not done because legacy artifacts and geometric helpers use both.
- MFV `r` choice: `r` is nonnegative and weights the up-type spurion only in `C_Q = r YuYu^\dagger + YdYd^\dagger` (`quarkConstraints/model.py:65`, `quarkConstraints/model.py:78`, `quarkConstraints/model.py:235`). Library scan default grid is `[0.10, 0.25, 0.40]` (`quarkConstraints/scan.py:177`), while the WQ 1M quark-only production grid is `(0.05, 0.10, 0.25, 0.50, 1.00)` across 10 M_KK tiles and 20k draws per `(r, M_KK)` (`scripts/wq_quarkonly_1m_plan.py:13`, `scripts/wq_quarkonly_1m_plan.py:15`, `scripts/wq_quarkonly_1m_plan.py:99`, `scan_outputs/wq_quarkonly_1M_20128400/scan_plan.json:19`). Rationale: span down-dominated, default-near, and up-dominated LH-doublet localization. Alternative: fit/scan continuous `r`; not used for WQ 1M.
- Perturbative color coupling default: if `g_s_star` is omitted, mass-basis KK-gluon couplings use perturbative QCD at `M_KK`; this is a conservative default rather than a strong 5D coupling benchmark (`quarkConstraints/couplings.py:96`, `quarkConstraints/couplings.py:145`). Rationale: avoid hiding an extra model choice in the core coupling builder. Alternative: fixed `g_s*` benchmark; can be supplied explicitly.
- epsilon_K budget choice differs by layer: core default uses the central residual `|epsilon_exp - epsilon_SM|` if no override is passed (`quarkConstraints/deltaf2.py:764`, `quarkConstraints/deltaf2.py:799`), while K001 catalog HARD veto uses the documented loose edge of a BGS grouped-SM plus experiment plus SM-choice band (`flavor_catalog_constraints/primary/kaon/K001.py:16`, `flavor_catalog_constraints/primary/kaon/K001.py:61`, `flavor_catalog_constraints/primary/kaon/K001.py:328`). Rationale: catalog exclusions should include the documented uncertainty band; diagnostics keep central/tight/loose values. Alternative: central-only veto; retained in core diagnostics but not K001 hard veto.
- epsilon_K/deltaf2 normalization choice is deliberate: `me_vll = (2/3) f^2 m B`, Wilson `/6`, and direct `M12` in epsilon_K implement the half-times-one-third/color-Fierz convention; no additional factor of 2 is applied to epsilon_K (`quarkConstraints/deltaf2.py:342`, `quarkConstraints/deltaf2.py:722`, `quarkConstraints/deltaf2.py:733`, `quarkConstraints/deltaf2.py:764`). Rationale: Delta-mass observables compare to `2|M12|`; epsilon_K uses `Im M12 / Delta m_K`. Alternative: doubling epsilon_K would double-count and contradict tests.
- QCD constants convention: standalone `qcd/` uses `ALPHA_S_MZ = 0.1180` (`qcd/constants.py:7`), while the DeltaF=2 LO wrapper default is `0.1179` (`quarkConstraints/qcd_running.py:34`). Rationale: the wrapper keeps compatibility with existing DeltaF=2 audit tests (`tests/test_wilson_rg_audit.py:55`). Alternative: unify constants; buildable but would require reblessing audit numerics.

### 2. W8 PR2 loop contract and W9 scan policy

W9 does not implement W8 physics formulas. W9 only exposes the selected EW model through the scan harness.

W9 production decision: top-partner loops and new custodial-FCNC modes are deferred for this scan. The custodial scan command must not pass any W8 loop or FCNC kwargs unless a separate reviewed run plan explicitly changes this policy. The run and comparison metadata must state:

- `custodial_tree_status="included"`
- `custodial_top_partner_loop_status="deferred"`
- `custodial_fcnc_mode="pr1_minimal_offdiag"`
- `oblique_proxy_status` matching the final W8/PR1 custodial proxy labels

Exact W8 contract W9 must preserve if these knobs exist after W8 lands:

- Default kwargs preserve PR1 behavior: `include_top_partner_loops=False`, `top_partner_loop_components="defer"`, `top_partner_loop_t_sign=None`, `top_partner_loop_delta_t_override=None`, `top_partner_loop_mass_ratios=None`, `top_partner_loop_mixing_scales=None`, `custodial_fcnc_mode="pr1_minimal_offdiag"`, and `kappa_fcnc=0.0`.
- `include_top_partner_loops=True` remains invalid for `ew_model="minimal_rs"` and is allowed for `ew_model="custodial_rs_plr"` only through W8's validated path.
- Loop-enabled mode is not a single boolean. A loop-applied scan would need explicit component selection (`singlet`, `bidoublet_vertex`, or `singlet_plus_bidoublet_vertex`), explicit sign or numeric `top_partner_loop_delta_t_override` policy for any applied `Delta T_loop`, optional mass-ratio/mixing inputs, and separate metadata for Zbb loop application versus EW001 T-loop application.
- W9 must not silently convert missing W8 sign/model inputs into zero while claiming loop numerics were included. If the final W8 implementation raises on missing loop inputs, W9 lets that raise during smoke validation; if W8 computes but does not apply missing-sign magnitudes, W9 records the final metadata as deferred/not applied and still does not advertise loop-included physics.

- [R21-I2] severity:INFO tag:infra
  Title: `cloudflare-pages.config.md` is a doc, not a `wrangler.toml` or `cloudflare-pages.toml` — Cloudflare Pages dashboard config must be entered manually
  Description: The R21 prompt asks "Cloudflare config (wrangler.toml or similar)". The deployment config in this unit is a markdown _document_ (`flavor_catalog/website/cloudflare-pages.config.md`, 111 lines) that captures the dashboard values rather than a machine-readable `wrangler.toml` or `cloudflare-pages.toml`. Cloudflare Pages git-driven deployments do not _require_ a `wrangler.toml` (unlike Cloudflare Workers); the build command + output dir are entered in the Pages dashboard. The website pairs the doc with `.node-version`, `public/_redirects`, and `public/_headers`, which are the only machine-readable Cloudflare-Pages config files this stack uses. Pattern is correct for Cloudflare Pages but a fresh operator must manually replicate the dashboard fields from the `.md` rather than `terraform apply`-style. Not a defect; an awareness flag for future infra-as-code work.
  Evidence: `flavor_catalog/website/cloudflare-pages.config.md:14-25` (dashboard field table); `flavor_catalog/website/.node-version` (Node 22 pin); `flavor_catalog/website/public/_redirects:11-12`; `flavor_catalog/website/public/_headers:1-20`; `git ls-tree origin/flavor-catalog-website/2026q2 flavor_catalog/website/ | grep -i wrangler` returns nothing.
  Recommended fix: Optional — add a `wrangler.toml` for Pages (supported but not required) if/when the team wants the deployment fully reproducible from VCS without dashboard touchpoints. Out of scope for R21.

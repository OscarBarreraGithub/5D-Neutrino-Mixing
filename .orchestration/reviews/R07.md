## Check 1 — Physics (Wilson UL bound only)

**Verdict**: PASS
**Literature anchors**: Wilson (1927); Brown/Cai/DasGupta (2001) for Wilson vs Clopper-Pearson at k=0; Csaki-Falkowski-Weiler 0804.1954 (KK tower); Carena-Ponton-Santiago-Wagner hep-ph/0701055 (EWPO floor).
**Evidence**:
- Wilson UL formula at `quarkConstraints/finite_stats.py:24-26`: `(z² + 2k + z·√(z²+4k(1-k/n))) / (2(n+z²))`. Algebraically equivalent to the canonical `(p̂ + z²/(2n) + z·√(p̂(1-p̂)/n + z²/(4n²))) / (1 + z²/n)` (multiply num+denom by `2n`). Verified by hand at k=0 reduces to `z²/(n+z²)`.
- Default `z=1.92` (`finite_stats.py:8`) is an explicit audit-convention choice documented at `docs/audits/zero_pass_inventory.md:13-14` and `docs/phase_logs/phase2_h8_review.md:7`. The choice makes `z² = 3.6864 ≈ −ln(0.025) = 3.6889`, so the Wilson UL at z=1.92 closely matches the **two-sided Clopper-Pearson 95% upper endpoint** at large n (3.6864/n vs 3.6889/n; difference < 0.07%). For n=10000: code yields `3.685e-4`, scipy `binomtest(0,n).proportion_ci(method='exact')` yields `3.688e-4`, matching the prompt's quoted ≈3.69e-4 target. Standard Wilson z=1.96 would give 3.84e-4 and would *not* match the published numbers in the paper.
- Scope appendix (`docs/quark_scan_methodology_note.tex:1143-1194`) lists four assumption blocks: (i) KK tower truncation = first colour-octet mode only with a quoted +20-30% systematic; (ii) EWPO floor is external (not computed); (iii) `g_s*=3` headline convention with `g_s≈1.05` perturbative default and `g_s≈6` only for CFW comparison; (iv) lepton sector out-of-scope. All four match the code: scan uses first KK gluon (`quarkConstraints/paper_0710_1869/couplings.py:69` is the only "first-mode" guard), `scanParams/scan.py` does not compute oblique parameters, and `quarkConstraints/scan.py:378-379` defaults to perturbative `g_s_star=None`.
- Zero-pass UL claims in `docs/quark_scan_methodology_note.tex:748-750, 766-771, 865-868` are cross-listed in `docs/audits/zero_pass_inventory.md` rows with N, k=0, gate definition, c-pattern, Y-prior, and output hash — all internally consistent.

# C15 — Backfill `.orchestration/pytest_selection/<unit>.txt`

**Tier:** 5 · **Lane:** det · **Issues closed:** R05-I2 (LOW, tag:infra) · **Status:** DONE

## Scope

R05-I2 observed that `.orchestration/pytest_selection/` was empty: the
orchestrator-side contract in `MERGE_PLAN.md:332` (and §B.3) requires one
selection file per review unit, listing the `tests/*` files touched by that
unit's commits, so a resumed reviewer can rerun the exact pytest selection.
R05 fell back to a `-k` keyword filter; the gap likely applied to every
Phase-1 review unit. C15 retroactively generates the 24 missing files
(R01–R09, R10a/b/c, R11–R22).

This is a `.orchestration/`-only artifact backfill. No source code, no tests,
no docs, no notebooks, no figures touched.

## Implementation

Single Python driver `/tmp/c15_generate.py` (one-shot; not checked in):

1. Hand-transcribed the unit → commit-list mapping directly from
   `MERGE_PLAN.md §B.1` (lines 292–315). The annotation on line 312 — `82a96f0`
   belongs to R03 only and is intentionally *not* in R19 — was honored:
   R19 receives 33 SHAs, R03 receives 4 (including `82a96f0`).
2. For each SHA, ran `git show --name-only --pretty=format: <sha>` and
   collected lines beginning with `tests/`.
3. For each unit, wrote sorted-unique test paths to
   `.orchestration/pytest_selection/<unit>.txt`. Units that touched no test
   files received a single line: `# no test changes in this unit`.

One MERGE_PLAN typo fixed inside the driver (MERGE_PLAN.md itself untouched):
`ebdo66c` (R18) → `ebd066c` (verified via `git log` as the PKA-B013 draft).
Five other SHAs cited in MERGE_PLAN.md could not be resolved against current
objects, reflog, or unreachable objects: R08 `e0b4e2`, R15 `7500919`, R18
`e9f3cf3`, R19 `1cf8b57`, `cd3a8fe`. All five sit in catalog/figure-only
units whose resolved SHA sets already touch zero `tests/`, so the selection
content is unaffected. Flagged in Follow-ups below; not in C15 scope.

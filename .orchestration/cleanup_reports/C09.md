## Forward-looking notes

- The new `unsnapshotted_reason:` field and `status: UNSNAPSHOTTED-BY-POLICY` value at `citation_anchors/B025.yaml` set a convention for documenting future no-PDF-policy gaps. If more such gaps surface (e.g. future conference-talk inputs to HFLAV averages), the same pattern can be re-used: a top-level `sources` entry pointing at the closest text-snapshotted parent document, with the talk PDF gap captured in the `unsnapshotted_reason` block and `status: UNSNAPSHOTTED-BY-POLICY` on the per-anchor record.
- The B009 `status_history` `state:` field is formally redundant with the `to:` field (both carry the same value when present). The catalog could either (a) standardize on `state:` as the canonical key and remove `to:` from all entries, or (b) standardize on `to:` and drop the redundant `state:`. C09 picks the minimum-touch path of restoring `state:` on the single entry that lacked it, matching the convention used by all other entries in the same yaml. A future schema-tightening pass could pick a single canonical key if desired.

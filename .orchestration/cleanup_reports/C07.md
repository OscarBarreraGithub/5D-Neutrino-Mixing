# C07 — per-master-compile annotations + aggregate fact-check script

**Tier:** 3 · **Lane:** opus · **Issues closed:** R17-I2, R18-I3, R19-I4 (all LOW, tag:docs) · **Status:** DONE

## Scope

Three LOW issues in `.orchestration/ISSUES.md` flag the same recurring pattern:
the consolidated `flavor_catalog/audits/factcheck_status.md` table was not
regenerated at each master-compile bump, so the per-master-compile reports
(`master_compile_v0{2,3,4}_report.md`) cite a headline VERIFIED count that
does not match the number of rows in the consolidated table at that revision.
The Wave-7 PRIMARY and Wave-8 SECONDARY verdicts that close the gap live in
per-family addendum sections rather than the consolidated table.

C07 is a docs-only cleanup. It annotates each compile report with a
"Consolidation status" block that reconciles the headline count against
the per-family audits at that boundary, and lands a small read-only
aggregation script for future master-compile cycles.

Per the cleanup plan (G3 = lean B), this is the per-master-compile
annotation + optional aggregate script disposition.

## Part 2 — optional aggregate script

`tools/aggregate_factchecks.py` (new file, ~155 lines, Python 3.8+ stdlib
only) scans `flavor_catalog/audits/factcheck_status.md` plus every
`factcheck_<family>.md` and emits per-family + overall VERIFIED / PARTIAL
/ MISMATCH / FAILED counts. It is forward-looking: future master-compile
runbooks can use it to regenerate the consolidated table or at least
spot-check the headline arithmetic in each `master_compile_v0X_report.md`.

Design notes:

- Recognises two row shapes used by the catalog:
  (a) markdown table rows whose first column is a process ID like
  `| K001 | kaon | VERIFIED | ...` (consolidated table + Wave-8 SECONDARY
  summaries), and (b) section headers like `## B012 - VERIFIED` and
  `### T003 - VERIFIED` (Wave-7 PRIMARY addenda sometimes use the
  header-only form).
- Precedence is: per-family files first, then the consolidated
  `factcheck_status.md` table overrides — so corrections folded into the
  consolidated table (e.g. T020 MISMATCH → VERIFIED after the follow-up
  fix) are respected.
- Read-only; the script does NOT edit any catalog file.
- Two output modes: human-readable text (default) and `--json`.

# C14 — codex model version drift fix (R14-I3 LOW, R20-I2 INFO)

**Unit**: C14 (Tier 5, opus lane)
**Closes**: R14-I3 LOW, R20-I2 INFO
**Date**: 2026-05-26
**Lane**: docs-only; no code touched; no test re-run required.

## Problem

Multiple catalog orchestration docs cited `gpt-5.5` / `GPT-5.5` as the codex
model, while the user's global instructions (`~/.claude/CLAUDE.md` Agent
Orchestration section) and the actually-configured `~/.codex/config.toml`
pin the model at `gpt-5.4`. A fresh handoff Claude reading both surfaces
saw two model names. Version-string inconsistency only; no catalog
content affected.

## Resolution

Replaced `gpt-5.5` / `GPT-5.5` → `gpt-5.4` / `GPT-5.4` in all
prescriptive / forward-looking catalog docs. Preserved the string in
historical / provenance contexts where it records what was actually used
at the time (any rewrite there would falsify the historical record).

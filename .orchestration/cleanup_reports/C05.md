## Implementation

1. **`sha256sums.txt`** — generated via
   `sha256sum deepresearch_may15.pdf deepresearch_may15.txt deepresearch_may15_review.md deepresearch_may16.pdf deepresearch_may16.txt deepresearch_may16_review.md > sha256sums.txt`.
   Six rows. `MANIFEST.md` and `sha256sums.txt` itself are intentionally
   excluded so the manifest can be edited without invalidating the list.

2. **`MANIFEST.md`** — added. Distinguishes "imported artifact" vs "review
   memo" per task spec. Each row records: filename, kind, source, subject,
   date imported (from `git log -1 --format=%ci` on the original import
   commits `022a20c` for the PDFs/txts, `2c00d84` for may15 review,
   `8fb5f91` for may16 review), file size, sha256, description (drawn from
   the existing review-memo headers and Wave-7 worklog), extraction
   provenance, session URL (recorded as not-captured-at-import per R17-I1
   sub-item (b)).

3. **Closing notes** in MANIFEST address each of R17-I1's three sub-items
   explicitly: (a) closed by digests; (b) partially closed — session URLs /
   prompts not retroactively recoverable, with a forward-looking convention
   for future imports; (c) partially closed — `.txt` extracts treated as
   best-effort mirrors with a recipe-recording convention for any future
   re-extraction.

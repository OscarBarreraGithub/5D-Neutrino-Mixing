# STATE OF PROJECT

## EXECUTIVE SUMMARY

This project is a 5D warped/RS flavor scan stack: it has geometry and wavefunction bookkeeping, quark MFV fits, KK-gluon DeltaF=2 matching, RS electroweak/Zbb/custodial modules, a broad flavor/collider catalog, a production scan harness, and a web-facing Scan Explorer. The headline quark-only result (post-audit; see `docs/FLOOR_SUMMARY.md` and `reports/collaborator_2026-06/CONTENT.md`) is that the minimal non-custodial model has a TYPICAL (median anarchic) physical-M_KK floor of ~30 TeV set by the TUNABLE flavor constraint `epsilon_K` (K001), and an EXISTENCE (fine-tuned best-point) floor of ~18-20 TeV set IRREDUCIBLY by oblique S,T,U (EW001) -- the RS T-problem, which carries no Yukawa freedom. The corrected `Z->bb` / T010 constraint collapses to ~5 TeV and is NOT the driver: the old "25-30 TeV Z->bb-dominated floor" (and the "108 TeV at 1sigma") was a B1 sign/normalization artifact in the Casagrande fermion-KK admixture, now fixed and dual-reviewed. The custodial P_LR tree-level branch fixes the oblique T problem but does NOT relax `epsilon_K`; its strict floor is about 2-3 TeV from rigorous DeltaF=2 constraints and the custodial inclusive floor is about 7 TeV once proxy EW001/collider failures are also counted (`.orchestration/PHASE2_PROGRAM_LEDGER.md:75`). The M_KK axis in these scans is the physical first gauge KK mass, approximately `M_KK = 2.45 * Lambda_IR`; do not explain the (now-retracted) Z->bb floor as merely a Lambda_IR-vs-physical convention conversion -- it was an actual code bug. What is rigorous at a glance: the geometry/wavefunction base, Bessel spectrum solver, quark MFV core, mass-basis KK-gluon couplings, the core audited DeltaF=2 constraints, minimal Zbb/T014 RS-EW plumbing, tree-level custodial P_LR diagonal protection, L001's typed LMFV carrier path, and the strict/proxy scan tagging. What is approximate at a glance: LO DeltaF=2 Wilson evolution, some hadronic inputs, C002 charm CPV, radiative B proxies, EW001 oblique prediction, collider recast proxies, nonleptonic/EDM partials, deferred custodial top-partner loops, and the unscanned lepton sector. What was actually scanned was not the full catalog: it was two paired 1M-row quark-only WQ scans, minimal `20128400` and custodial `20675555`, on the same `r x M_KK` grid, with the swept lepton sector dropped (`scripts/run_full_catalog_scan.py:73`, `scan_outputs/wq_quarkonly_1M_20128400/analysis/analysis_report.md:3`, `scan_outputs/wq_quarkonly_1M_custodial_20675555/analysis/analysis_report.md:3`). Strict survival is the defensible physics floor because it ignores proxy HARD failures; inclusive survival is a sensitivity display that also counts proxy HARD failures (`scripts/run_full_catalog_scan.py:930`, `scripts/run_full_catalog_scan.py:932`, `scripts/run_full_catalog_scan.py:933`). The single most important open item is a full with-lepton production scan lane, with live lepton-sector ParameterPoint inputs and all HARD lepton constraints, because the current floors are strong quark-sector results but not final full-catalog exclusions (`scripts/run_full_catalog_scan.py:75`, `scripts/run_full_catalog_scan.py:521`).

## APPROXIMATE

### Geometry

- The main geometry implementation is rigorous, but exported artifacts must keep the two scale conventions explicit: core geometry treats `Lambda_IR` as `1/z_v`, while scans quote physical first-gauge-KK M_KK. Mixing these conventions would misstate floors by the `x_1 ~= 2.45` factor (`quarkConstraints/scales.py:20`, `quarkConstraints/scales.py:21`, `quarkConstraints/scales.py:33`, `quarkConstraints/scales.py:49`, `scripts/run_full_catalog_scan.py:145`, `scripts/run_rs_anarchy.py:85`).

### Quark Flavor / DeltaF=2

- DeltaF=2 Wilson RG is LO, not at the same rigor level as the 4-loop `qcd/` alpha_s package. ### Geometry

- Scale convention: core geometry treats `Lambda_IR` as `1/z_v`; physical first gauge KK mass is `2.448687... * Lambda_IR`. Quark bookkeeping default is `M_KK = xi_KK * Lambda_IR` with `DEFAULT_QUARK_XI_KK = 1.0`, while publication/full-catalog scans use `xi_KK = 2.4487` (`quarkConstraints/scales.py:20`, `quarkConstraints/scales.py:21`, `quarkConstraints/scales.py:33`, `quarkConstraints/scales.py:49`, `scripts/run_full_catalog_scan.py:145`, `scripts/run_rs_anarchy.py:85`).

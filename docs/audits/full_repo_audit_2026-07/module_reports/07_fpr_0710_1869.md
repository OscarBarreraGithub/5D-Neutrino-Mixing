### [CRITICAL] Wilson coefficients are RG-evolved with the operator-evolution matrix (inverted direction + missing transpose)
- **File:** paper_0710_1869/eft_deltaf2/rg.py:288-313 (`_closed_form_bmu_lr_segment_matrix`), rg.py:1361-1365 (`vll_factor`/`vrr_factor`), applied at rg.py:1516-1519
- **Category:** wrong-formula / sign
- **Claim:** LO evolution uses `U = η^{+γ₀/(2β₀)}` with η = α_s(μ_low)/α_s(μ_high) and the *untransposed* ADM — the matrix that evolves operator matrix elements, not Wilson coefficients. Correct is C(μ_l) = [α_s(μ_h)/α_s(μ_l)]^{γ₀ᵀ/(2β₀)} C(μ_h) (BBL eq. 3.94), i.e. the inverse-transpose of what the code applies.
- **Evidence:** Numerical run 3 TeV → 2 GeV gives paper-basis evolution diag(1.372, 1.372) for Q1_VLL/VRR and LR block [[0.282, 0],[1.185, 1.171]] (C4→0.28·C4, C4 feeds C5). Physics requires Q1_VLL **suppressed** ≈ 0.73 and SUSY-basis **C4 enhanced** ≈ 3.5-5× with C5 feeding C4 — the inverse-transpose, [[3.54, −3.59],[0, 0.85]]. Code thus *suppresses* the chirally-critical C4 ~12× where it should enhance ~3.5×, and inflates C1 by 1.37 where it should shrink to 0.73. Every ε_K/ΔM number downstream is wrong by O(2-40) per operator.
- **Confidence:** high (verified by direct execution + closed-form ODE solution)
- **Fix:** Use `eta = alpha_start/alpha_end` (or negate exponents) and transpose the LR mixing (u21 = γ12/(γ11−γ22)(η_a1−η_a2)).

### [CRITICAL] Default ε_K path cannot contain the dominant LR (C4) contribution — the ~2 TeV floor is untestable by this pipeline
- **File:** paper_0710_1869/eft_deltaf2/matching_kkgluon.py:191-198, 504-527; kkgluon.py (`right_down` matrix)
- **Category:** logic-bug / wrong-formula (structural)
- **Claim:** The frozen "supported observable" subset is Q1_VLL/Q1_VRR only, and the default `right_down` KK-gluon matrix is exactly diagonal, so `right_coupling = 0` identically and C4 = C5 = C1_VRR ≡ 0 by construction.
- **Evidence:** Executed `default_paper_0710_1869_kaon_matching()`: `right = 0j`, `C4 = 0j`. Frozen contract hard-errors if `supported_observable_operator_names` differs from the Q1 subset (lines 285-291). But ε_K in RS is dominated by chirally-enhanced C4; FPR's V5KM alignment suppresses only the LL sector (MODEL_CONVENTIONS §4 itself notes this). Exact RH-down diagonality is a *stronger* assumption than FPR makes. Any "~2 TeV ε_K floor verified" claim from this default path validates only the subdominant LL operator.
- **Confidence:** high (structural; executed)
- **Fix:** Wire nonzero RH-down mixing and the combined Q1+LR observable surface into the default path before quoting a Lane-C ε_K floor.

### [MAJOR] Fierz sign error in the frozen paper↔BMU LR basis map (+2 should be −2)
- **File:** paper_0710_1869/eft_deltaf2/rg.py:110-125 (`_PAPER_TO_BMU_LR_OPERATOR_MAP`)
- **Category:** sign / paper-transcription
- **Claim:** Frozen map asserts Q1^LR_BMU = +2·Q5_paper; the standard Fierz identity gives Q1^LR = **−2** Q5 (and C1^LR_BMU = −½C5, not +½C5).
- **Evidence:** (ψ̄₁P_Lψ₂)(ψ̄₃P_Rψ₄) = −½(ψ̄₁γ^μP_Rψ₄)(ψ̄₃γ_μP_Lψ₂). Cross-check: correct BMU matching gives C1^LR = −g_Lg_R/(6M²); code's map applied to its (correct) C5 yields +g_Lg_R/(6M²). Map applied twice so diagonal running unaffected, but C5→C4 mixing contribution (and any BMU-basis observable) flips sign — interference-sign error in ε_K.
- **Confidence:** medium-high
- **Fix:** Change operator map 2 → −2 (and Wilson maps/inverses correspondingly).

## How to read this document

Findings are grouped into: (0) executive summary, (1) critical, (2) major, (3) cross-cutting systemic themes, (4) minor findings by module (condensed), (5) things checked and found **correct** (equally important for a physics audit), (6) coverage — what was *not* reviewed. Every finding cites `file:line`, and the per-module reports contain the full evidence chains (quoted code, re-derivations, numerical check outputs).

Severity meaning:
- **CRITICAL** — invalidates a quoted headline result or produces silently wrong physics output at O(1) or worse.
- **MAJOR** — a real physics/logic/statistics error with material (≳10%) or structural impact.
- **MINOR / NOTE** — real but small-impact defects, stale data, latent footguns, coverage gaps.

The geometry/wavefunction base, the SVD/Takagi linear algebra, the post-audit ΔF=2 core matching (CFW color/Fierz factors, GGMS matrix elements, ε_K master formula), the 4-loop α_s machinery, the Z-pole formula layer, and most experimental anchors in the catalog are **verified correct** by independent re-derivation. The serious problems cluster in five places:

4. **Lane A (anarchic reproduction) carries a factor-2 convention error**: the Bauer→repo Yukawa bridge cancels only one of two √2's, so the ensemble is effectively Bauer's prior with Y_max ≈ 1.5 (not 3), inflating ε_K^NP ~4× and biasing the quoted ~10/~30 TeV anarchic walls high by ~2×; plus a missing Wolfenstein-A in f_Q1 and non-reproducible `hash()`-based seeds.

## 1. CRITICAL findings

### C-1. Lane C RG evolution inverted (direction) and untransposed (basis)
`quarkConstraints/paper_0710_1869/eft_deltaf2/rg.py:288-313, 1361-1365, 1516-1519` — the LO evolution applies `U = η^{+γ₀/(2β₀)}` with η = α_s(μ_low)/α_s(μ_high) and the untransposed ADM: this evolves *operators*, not Wilson coefficients. Verified by direct execution: 3 TeV→2 GeV gives C1×1.372 (should be ×0.73) and C4×0.282 with C4→C5 feed (should be C4×~3.54 with C5→C4 feed). Correct: C(μ_l) = [α_s(μ_h)/α_s(μ_l)]^{γ₀ᵀ/(2β₀)} C(μ_h) (Buchalla-Buras-Lautenbacher Eq. 3.94). Downstream ε_K/ΔM numbers wrong by O(2–40) per operator. **Note:** `quarkConstraints/qcd_running.py` (used by Lane B and `modern/`) was independently verified to do this *correctly* — the bug is isolated to the `paper_0710_1869` lane. → [Report 07](module_reports/07_fpr_0710_1869.md), cross-check in [Report 08](module_reports/08_modern.md).

### C-2. Lane C default ε_K path structurally zeroes the dominant LR operator (C4 ≡ 0)
`paper_0710_1869/eft_deltaf2/matching_kkgluon.py:191-198, 504-527` + `kkgluon.py` — the frozen observable contract admits only Q1_VLL/Q1_VRR, and the default RH-down KK-gluon coupling matrix is exactly diagonal, so `right_coupling = 0` and C4 = C5 = C1_VRR ≡ 0 identically (verified by execution). ε_K in RS is dominated by the chirally-enhanced C4; FPR's V5KM alignment suppresses only the LL sector. Any "Lane C ε_K floor" from this pipeline tests only the subdominant operator. Combined with C-1 and the ξ_g=1 default (M-13), **the ~2 TeV Lane-C floor must not be quoted as code-verified.** → [Report 07](module_reports/07_fpr_0710_1869.md).

### C-3. EW001 oblique anchors match no published fit, and the implemented floor is ~16 TeV, not 18–20 TeV
`flavor_catalog/processes/top_higgs_ew/EW001.yaml:82-118` — active HARD ellipse S=+0.026±0.075, T=+0.047±0.066, ρ=0.90, labelled "PDG 2025 Table 10.8". PDG 2022 (U-fixed): S=−0.01±0.07, T=+0.04±0.06; PDG 2024: S=−0.05±0.07, T=0.00±0.06. The positive S central matches no edition (two independent reviewers concur). Solving the shipped χ²(2 dof, 5.99) with the shipped coefficients (c_S=30, c_T = x₁²πL/(2c_W²) = 428.8 at L=35): **M_KK floor = 15.96 TeV** (custodial: 5.80 TeV), vs the documented lane-independent "existence ~18–20 TeV" (`FLOOR_SUMMARY.md:25`, `CLAUDE.md`). The stale positive T central is anti-conservative (RS predicts ΔT>0). Partially tracked in `docs/audits/oblique_stu_pdg_staleness.md`, but that doc's own update concedes the anchors are now floor-driving. → [Reports 05](module_reports/05_rs_ew.md), [11](module_reports/11_catalog_ew_collider_lepton.md).

### C-4. KK Bessel solver silently skips and misorders tower roots (n ≳ 5)
`solvers/bessel.py:153-179, 264-285` — brackets use a ±20% *relative* window around asymptotic seeds; beyond x≈15 this exceeds the ~π root spacing, so `brentq` lands on arbitrary roots. Verified against a brute-force sign scan of the solver's own quantization function: gauge NN skips the 6th root (18.118); fermion ++ (c=0.27) returns a 5th "mass" of **52.98 instead of 15.29** (×3.5 off) with an unsorted array. No warning fires. First 3–4 roots (the default `n_roots=3`) are correct in all tested cases — so current production consumers are likely safe, but anything asking for a deeper KK tower gets silently wrong spectra. → [Report 01](module_reports/01_lepton_core.md).

5. **The certification/documentation layer lags the code** (M-22, M-24, and stale line references throughout `docs/audits/`); conversely the code sometimes lags its own decisions (`MODEL_CONVENTIONS.md` §3 mandates the affine bulk-mass map; production still routes through `BulkMassMap` — documented TODO).

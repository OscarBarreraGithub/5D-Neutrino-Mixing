## B021: Lambda_b^0 -> Lambda l+l-

### Verdict
APPROVE-OVERRIDE

### Cycle-3 CA issue
`CHK-1`: the TeX states LHCb used `3.0 fb^-1`, but that dataset
integrated-luminosity number remains only under
`supporting_measurements`, not promoted to `pdg_or_equivalent`.

### Rationale
The L001 precedent (`flavor_catalog/signoff/by_process/L001.md`) is on
point: `pdg_or_equivalent` is the canonical block for measured
experimental observables (branching fractions, angular asymmetries,
limits, masses, etc.), each carrying value, uncertainty, year,
source_url, access_date, and sha256. "Dataset integrated luminosity"
is not a measured observable; it is experimental-run metadata that
contextualizes the LHCb 2015 analysis already promoted under
`pdg_or_equivalent.observables` for the differential branching
fraction and A_FB^l, A_FB^h (all four blocks point to the same
`lhcb_1503_07138_lambdab_lambdamumu.txt` snapshot with sha
`2feb8efe212b`). Promoting `3.0 fb^-1` into `pdg_or_equivalent` would
extend the schema's measured-observable contract to cover every
auxiliary dataset descriptor (center-of-mass energy, trigger thresholds,
selection efficiencies, etc.), which CA's strict reading would then
force in every catalog entry.

The `3.0 fb^-1` value already has full provenance under
`supporting_measurements[1].values.integrated_luminosity`, with
`snapshot_path`, `sha256_of_text_snapshot`, `source_key`, and
`source_url` -- this is the schema-correct home for dataset metadata,
exactly parallel to how the CDF 2011 dataset descriptors (signal yield,
significance, luminosity, sqrt(s)) are now also reflected in
`pdg_or_equivalent.values` (where the CDF branching fraction lives)
while the same descriptors are summarized in `supporting_measurements`.

Note on the suggested mechanical fix: the prompt suggests renaming
`local_snapshot_path -> snapshot_path` in
`references/B021/source_manifest.yaml`, but inspection shows all five
entries already use `snapshot_path` plus
`sha256_of_local_snapshot`; no rename is needed. The cycle-3 CA
issues list contains exactly one item, the CHK-1 luminosity-promotion
issue handled above.

## Policy guidance for future CAs (beauty family)

- `pdg_or_equivalent` is the home for measured observables and their
  uncertainties. Dataset metadata (integrated luminosity,
  sqrt(s), event yields used only to contextualize an analysis) belongs
  in `supporting_measurements`. Theoretical normalization scales and
  reference points belong in `paper_era_reference`. This matches both
  the L001 precedent and the plan v1 Section B schema intent.
- Citation-key mismatches between TeX prose and `source_manifest.yaml`
  are real CHK-2 fails and should be patched at the typo level rather
  than escalated.

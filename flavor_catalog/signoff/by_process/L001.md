## Rationale
Section B of plan v1 defines `pdg_or_equivalent` as the canonical PDG-or-equivalent block for measured observables (value, uncertainty, year, source URL, access date, sha256). The `3 TeV` quantity in the Perez--Randall NDA formula `(3 TeV / M_KK)^4` is a theoretical reference-scale normalization choice, not a measured observable; promoting it into `pdg_or_equivalent` would misuse the schema and would require boilerplate provenance entries for every theory-input constant in every catalog file. The Perez--Randall paper itself is already traceable through `pdg_or_equivalent.paper_era_reference` (with source URL, access date, snapshot path, and sha256 to `flavor_catalog/references/L001/perez_randall_2008_arxiv0805_4652.txt`), which is the correct provenance anchor for the NDA-formula numerals. CA-v3's CHK-1 issue is therefore a too-strict reading for this case, and L001 is otherwise CHK-1 through CHK-8 PASS per `ca_w23_charged_lepton_v3.md`.

## Policy guidance for future CAs
CHK-1's strict "every numerical claim must have a `pdg_or_equivalent` value block" reading applies to measured experimental observables and to repository defaults whose value is asserted as a physical input (branching-ratio limits, masses, mixing parameters, lifetimes, CKM elements, etc.); it does NOT apply to theoretical normalization conventions, reference scales used inside derivation formulas, or NDA prefactors whose meaning is "this formula is written normalized to X" -- those belong inside the originating source block (here `paper_era_reference`) or in `code_coverage.evidence` rather than as standalone `pdg_or_equivalent` entries.

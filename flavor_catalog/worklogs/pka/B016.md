# PKA Worklog: B016

Process: `B -> K ell+ ell-` exclusive branching fractions.

I first checked the required catalog plan, orchestrator decisions, Opus
approval, process template, and README, then verified that no B016 process files
or reference directory already existed.  I used the nearby B017/B018 entries
only as schema and style examples; B016 has its own snapshots and source
manifest.

Primary value search focused on HFLAV rare-B tables because the process row
names HFLAV/LHCb/Belle and asks for the central exclusive `b -> sll`
branching-fraction observable.  Direct HFLAV URLs checked:
`radll/Bu/BR_B+_K+_l+_l-.html` and
`radll/Bd/BR_B0_K0_l+_l-.html` under the Dec2025 HFLAV EOS area.  The charged
mode gives `(5.76 +/- 0.40)e-7`; the neutral mode gives `(3.28 +/- 0.32)e-7`.
Both snapshots record HFLAV's note about differing charmonium treatment among
inputs.  I also saved minimal arXiv metadata/abstract snapshots for Belle
`1908.01848`, LHCb `1403.8044`, BaBar `0807.4119`, and the RS-flavor baseline
`0804.1954`; no publisher PDFs were downloaded.

Rejected/secondary sources: I did not use the Springer or APS publisher pages
directly, because the catalog policy prefers arXiv text snapshots and minimal
source excerpts.  I also did not promote LHCb/Belle `R_K` numerical values into
B016; those belong in companion B018, while B016 is the branching-fraction row.

Code coverage was rerun across `quarkConstraints`, `qcd`, `flavorConstraints`,
`neutrinos`, `yukawa`, `warpConfig`, `solvers`, `scanParams`, and `tests` with
the plan greps plus targeted searches for `B_to_K`, `BtoK`, `b -> s ell`,
`C9`, `C10`, and `R_K`.  Results show existing Delta F = 2 and mu -> e gamma
support only; no rare semileptonic B backend was found.  I classified
implementation difficulty as HIGH because a useful constraint needs a new
Delta B = 1 semileptonic Hamiltonian, q2-binned predictions, hadronic inputs,
nonlocal charm treatment, and covariance handling.

Open issues: none blocking for PKA.  WA/CA may decide whether to keep charged
and neutral modes in one B016 entry or split them in a later catalog pass.

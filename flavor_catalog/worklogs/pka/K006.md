# PKA Worklog: K006

I identified the assigned process as `K006`, `K_L -> mu+ mu-`, from the kaon-sector table in plan v1 Section C.  The family is `kaon`; there was no ambiguity in the row once the process description mentioned the long-distance dominated dimuon mode with a short-distance dispersive constraint.

For the canonical value I used the PDG 2024 `K0_L` listing, because it gives the current catalog value for `Gamma_24 mu+ mu- S1`: `BR(K_L -> mu+ mu-) = (6.84 +/- 0.11) x 10^-9`.  I captured only a short text excerpt of the PDG listing and did not store a PDG PDF.  I also recorded the primary E871 paper through the PubMed/DOI metadata and abstract; that excerpt gives the normalization ratio and the primary branching-fraction measurement.  For short-distance interpretation I used Isidori-Unterdorfer, arXiv:hep-ph/0311084, because it gives the conservative `B_short < 2.5 x 10^-9` bound that is directly relevant to using this channel as an `s -> d` new-physics constraint.

For post-2008 developments I consulted D'Ambrosio-Kitahara 2017 on direct CP violation and `K_L`/`K_S` interference, Dery-Ghosh-Grossman-Schacht 2021 on time-dependent extraction of a clean short-distance observable, and Chao-Christ 2024 on a lattice-QCD framework for the two-photon long-distance amplitude.  I treated these as theory developments rather than replacements for the PDG branching fraction.

Code coverage was checked with a targeted `rg` over `quarkConstraints/`, `qcd/`, `flavorConstraints/`, `neutrinos/`, `yukawa/`, `warpConfig/`, `solvers/`, `scanParams/`, and `tests/`, excluding notebooks and binary-like files.  I found kaon `Delta F=2` mass-difference and test-observable hits, but no `K_L -> mu+ mu-`, dimuon rare-kaon, or `s -> d l+l-` implementation.  I therefore marked coverage as `NO` and implementation difficulty as `HIGH`: a production constraint would need new semileptonic rare-kaon matching plus a long-distance subtraction or conservative dispersive bound.

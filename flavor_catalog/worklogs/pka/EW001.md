# PKA worklog: EW001

Assigned process: `S/T/U` oblique electroweak parameters, process id `EW001`,
family `top_higgs_ew`.  I first checked out `flavor-catalog/2026q2`, then read
plan v1 sections B, D, and F, the orchestrator decisions file, and the local
process template.  The orchestrator document explicitly lists EW001 as a
NIT-1 accepted addition, so I did not look for a plan-v1 row.

For canonical values I used the current PDG electroweak review, "Electroweak
Model and Constraints on New Physics", revised November 2025 and accessed on
2026-05-16.  I stored only a small text snapshot, not the PDG PDF.  The
sidecar records the PDG U-fixed and floating-U STU fits, including central
values, uncertainties, and the main correlation metadata.  I also captured
the PDG warped-extra-dimension context, where the review translates an
S-parameter bound into an approximate KK-gauge scale bound.

I searched for recent global-fit context using queries for "PDG oblique
parameters S T U 2025", "GFitter oblique parameters", and "HEPfit CDF W mass
S T U".  The accepted auxiliary sources are the Gfitter public oblique-results
page, the HEPfit precision-electroweak page, and de Blas et al.
arXiv:2204.04204 for the CDF W-mass impact.  I rejected generic model papers
that merely reused STU inputs without being a global EW fit, plus publisher
PDFs and secondary aggregator pages.

For post-2008 context I used arXiv:0804.1954 as the requested CFW-era baseline.
The draft emphasizes that post-2008 changes are mostly global-fit and
input-theory changes: Higgs/top inputs, modern SM EW calculations, and the CDF
W-mass scenario.

Code coverage greps over `quarkConstraints/`, `qcd/`, `flavorConstraints/`,
`neutrinos/`, `yukawa/`, `warpConfig/`, `solvers/`, `scanParams/`, and `tests/`
found no S/T/U, oblique, EWPO, or W-mass likelihood implementation.  Only
generic electroweak constants and VEV comments surfaced, so I marked coverage
`NO` and difficulty `HIGH`.  The main unresolved policy choice is whether a
future implementation should expose a PDG-default likelihood only or also a
separate CDF-W-mass scenario switch.

# PKA Worklog: C005

Process: `D0 -> e+ e-` rare leptonic charm decay.

I first checked out `flavor-catalog/2026q2` and read the required planning
documents, Opus signoff, process template, and catalog README in order.  The
assigned row identifies C005 as a charm-sector rare dilepton FCNC with initial
coverage `NOT-SEEN`.

Source search focused on PDG Live/API, Belle, BaBar, and the RS baseline.
Queries included `PDG D0 e+ e- branching fraction upper limit`, `PETRIC 2010
Search for Leptonic Decays of D0 Mesons arXiv`, and `BaBar Search for the
decay modes D0 e+e- D0 mu+mu- D0 e mu arXiv`.  The canonical source is PDG
Live/API `S032.39`, accessed on 2026-05-16.  It gives
`BR(D0 -> e+ e-) < 7.9e-8` at 90% CL and marks the Belle 2010 result as used
in the average/fit.  Belle 2010 (`arXiv:1003.2345`) confirms the same limit
from `660 fb^-1`.  BaBar 2012 (`arXiv:1206.5419`) is retained as an
independent post-2008 search with the weaker `1.7e-7` limit from `468 fb^-1`.
The dispatch note called this an LHCb upper limit, but the accessed PDG API
measurement list contains no LHCb `D0 -> e+ e-` result; I recorded this as an
open issue for WA/CA instead of changing the process scope.

Snapshots are minimal text extracts only under `flavor_catalog/references/C005`;
no publisher PDFs were stored.  I added PDG, Belle, BaBar, CFW 2008, and a rare
charm theory-context snapshot from Burdman et al.  The CFW and theory entries
are kept outside `pdg_or_equivalent` in the YAML per the L001 arbitration
policy.

`flavorConstraints/`, `neutrinos/`, `yukawa/`, `warpConfig/`, `solvers/`,

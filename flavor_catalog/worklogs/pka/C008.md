# PKA Worklog: C008

Assigned process: `C008`, charm family, \(D^+ \to \pi^+ e^\pm \mu^\mp\).
I checked out `flavor-catalog/2026q2` first, then read the plan, orchestrator
decisions, Opus signoff, process template, and catalog README in the requested
order.  The plan row identifies C008 as a PDG/LHCb charged-lepton-flavor-
violating semileptonic charm mode with expected `NOT-SEEN` code coverage.

Source extraction found that PDG Live/API splits the process into two charge
modes.  S031.110 lists \(D^+\to\pi^+e^+\mu^-\) with a current 90% CL upper
limit \(<2.1\times10^{-7}\); S031.111 lists \(D^+\to\pi^+e^-\mu^+\) with
\(<2.2\times10^{-7}\).  Both current measurements are LHCb JHEP 06 (2021)
044 / arXiv:2011.00217, based on 1.6 fb\(^{-1}\) of 2016 \(pp\) data.  I
checked the arXiv source table directly; it gives companion 95% CL limits
\(2.3\times10^{-7}\) and \(2.2\times10^{-7}\).  I also retained BaBar
arXiv:1107.4465 as the post-2008 predecessor search with 90% CL limits
\(2.9\times10^{-6}\) and \(3.6\times10^{-6}\), using 384 fb\(^{-1}\).

I saved only minimal text snapshots: two PDG API extracts, LHCb 2021 table
extracts, BaBar 2011 table extracts, and a short arXiv:0804.1954 baseline
metadata note.  No publisher PDFs were downloaded or tracked.  All snapshots
are under 100 KB and are recorded in `source_manifest.yaml` and
`sha256sums.txt`.

For code coverage I ran the three catalog grep patterns over
`quarkConstraints`, `qcd`, `flavorConstraints`, `neutrinos`, `yukawa`,
`warpConfig`, `solvers`, `scanParams`, and `tests`, plus focused searches for
\(D^+\pi^+e\mu\), rare semileptonic charm, and \(c\to u\) LFV terms.  No C008
implementation was found.  The only nearby hits are existing \(D^0\) mixing
and the unrelated \(\mu\to e\gamma\) dipole check.  Main follow-up for WA/CA:
decide whether a future recast should combine the two charge modes or preserve

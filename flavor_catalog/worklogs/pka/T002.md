# PKA Worklog: T002

Process: `t -> u Z`, assigned from plan v1 Section C as `T002` in the
`top_higgs_ew` family.  The row is unambiguous: "Top FCNC Z decay;
ATLAS/CMS/PDG."  I used the existing T001 entry only as a schema/style guide
and created process-local T002 snapshots.

I first read the required plan, orchestrator decisions, Opus signoff, process
template, and catalog README, then checked that the T002 deliverables did not
already exist.  For numerical inputs I used the PDG 2025 top-quark listing as
the canonical aggregator and ATLAS arXiv:2301.11605 as the current primary
Run-2 source.  The PDG listing gives the strongest current `t -> Z u` limits:
`< 0.062 x 10^-3` for the left-handed benchmark and `< 0.066 x 10^-3` for the
right-handed benchmark, both at 95% CL.  I also captured the CMS 2017
comparison result, arXiv:1702.01404, with `B(t -> Z u) < 0.022%`.

For theory context I used Aguilar-Saavedra hep-ph/0409342 for the classic SM
benchmark `B(t -> u Z) = 8 x 10^-17`, CFW arXiv:0804.1954 as the requested
2008-era warped-flavor reference, and Casagrande et al. arXiv:0807.4937 for an
RS-specific discussion of `t -> c(u) Z`.  Casagrande et al. is especially useful
because it says the minimal-RS `t -> u Z` rate is typically two orders below
`t -> c Z` and difficult to test at the LHC.  I did not track publisher PDFs;
all local reference files are short text snapshots with URLs, access date, and
sha256 entries in the manifest.

Code coverage was checked with the three plan greps plus a targeted grep for
`t -> u Z`, `t -> Z u`, `tuZ`, `tZu`, `tZq`, and FCNC terms across the requested
code directories.  I found no implementation of this observable.  The only
targeted code-directory hits were generic FCNC prose/comments in
`quarkConstraints/PAPER_0710_1869.md` and `tests/test_paper_couplings.py`, not
a top-FCNC limit or calculation.

Open questions for WA/CA: decide whether the left-handed ATLAS value should be
the headline because it is marginally stronger, or whether both ATLAS chiral
benchmarks should remain equally prominent.  A future implementation also needs
a convention choice, because the experimental headline is a tensor-SMEFT
benchmark while RS may generate vector-like `Ztu` couplings or correlated
contact effects.

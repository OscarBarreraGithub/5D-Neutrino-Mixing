# PKA Worklog: C004

Assigned process: `C004`, charm family, \(D^0 \to \mu^+\mu^-\).  I checked out
`flavor-catalog/2026q2` first, then read the planner v1 document, orchestrator
decisions, Opus signoff, process template, and catalog README.  The plan row
lists this as a rare \(c\to u\ell\ell\) FCNC test with expected `NOT-SEEN` code
coverage.

Source extraction found a currency update relative to the dispatch wording.  The
prompt described the LHCb upper limit as the most recent result, but PDG Live/API
S032.28 now includes the CMS 2025 search as the current strongest input.  I used
the PDG 2026 API limit `<2.1 E-9` at 90% CL as the canonical catalog value and
recorded the CMS public-page companion result `<2.4 x 10^-9` at 95% CL.  I kept
the requested LHCb result, arXiv:2212.11203 / PRL 131 (2023) 041804, as the
previous best limit: `<3.1 x 10^-9` at 90% CL with the PDG-noted 95% CL
companion `<3.5 x 10^-9`.

For Standard Model context I used Burdman, Golowich, Hewett, and Pakvasa,
arXiv:hep-ph/0112235, because it gives the long-distance two-photon scaling
used in many rare-charm discussions: approximately \(2.7\times10^{-5}\) times
\(\mathcal B(D^0\to\gamma\gamma)\), with an anticipated floor of at least
\(3\times10^{-13}\) under their assumptions.  I also saved a minimal snapshot
of the 2024 rare-charm EFT analysis by Gisbert, Hiller, and Suelmann for the
post-2008 global-Wilson-coefficient context, plus the CFW arXiv:0804.1954
baseline for RS/anarchic flavor framing.

Licensing policy was followed: no publisher PDFs are stored, and all source
files are short `.txt` snapshots below 100 KB.  The manifest records URL,
access date, and sha256 for every snapshot.

For code coverage I ran the three required catalog greps over
`quarkConstraints`, `qcd`, `flavorConstraints`, `neutrinos`, `yukawa`,
`warpConfig`, `solvers`, `scanParams`, and `tests`, plus a focused search for
D0 dimuon, rare charm, \(c\to u\), and \(C_{10,S,P}\) patterns.  Only D0 mixing
and the unrelated \(\mu\to e\gamma\) check were found, so C004 is marked `NO`.

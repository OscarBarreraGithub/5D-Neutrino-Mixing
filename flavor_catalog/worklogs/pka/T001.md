# PKA Worklog: T001

Process: `t -> c Z`, assigned from plan v1 Section C as `T001` in the
`top_higgs_ew` family.  The row is unambiguous: "Top FCNC Z decay;
ATLAS/CMS/PDG."  I used `top_higgs_ew` because the process is listed in the
Top, Higgs, and electroweak flavor block.

I first read the required plan, orchestrator decisions, Opus signoff, process
template, and catalog README.  I then checked that the target files did not
already exist.  For numerical inputs I used the PDG 2025 top-quark listing as
the canonical aggregator and ATLAS arXiv:2301.11605 as the current primary
Run-2 source.  The PDG listing gives the strongest current `t -> Z c` limits:
`< 0.13 x 10^-3` for the left-handed benchmark and `< 0.12 x 10^-3` for the
right-handed benchmark, both at 95% CL.  I also captured the CMS 2017
comparison result, arXiv:1702.01404, with `B(t -> Z c) < 0.049%`.

For theory context I used Aguilar-Saavedra hep-ph/0409342 for the classic
SM benchmark `B(t -> c Z) ~ 1 x 10^-14` and CFW arXiv:0804.1954 as the
requested 2008-era warped-flavor reference.  I also included Airen and
Franceschini arXiv:2601.14966 because it is directly RS-specific and discusses
post-LHC allowed `Ztc` rates and future collider reach.  I did not track any
publisher PDFs; all local reference files are short text snapshots with URLs,
access date, and sha256 entries in the manifest.

Code coverage was checked with the three plan greps plus a targeted grep for
`t -> c Z`, `t -> Z c`, `tcZ`, `tZc`, and FCNC terms across the requested code
directories.  I found no implementation of this observable.  The only non-doc
matches were generic FCNC prose/comments in `quarkConstraints/PAPER_0710_1869.md`
and `tests/test_paper_couplings.py`, not a top-FCNC limit or calculation.

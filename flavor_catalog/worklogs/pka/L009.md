# PKA Worklog: L009 tau -> 3 mu

I followed the required planning docs and template, then checked that no L009
draft or reference directory existed before creating new files.  Source search
started from PDG 2025 tau listings and the current experimental papers for
`tau -> mu mu mu`.  Queries included "PDG tau mu mu mu branching fraction
limit", "Belle II tau to three muons 2024", "LHCb tau to three muons Run 2
2026", "Hayasaka tau decays into three leptons 2010 arXiv", and "BaBar tau
three charged leptons 2010 arXiv".

The canonical catalog value is the PDG 2025 tau listing value
`BR(tau^- -> mu^- mu+ mu-) < 1.9e-8` at 90% CL, attributed by PDG to the
Belle II ADACHI 24R result.  I also recorded the Belle II arXiv/JHEP source
for the same limit and the newer LHCb Run 2 preprint, which sets an equal
observed 90% CL limit and a 95% CL limit of `2.3e-8`.  Historical source
snapshots cover Belle 2010 (`2.1e-8`), BaBar 2010 (`3.3e-8`), and LHCb 2015
(`4.6e-8`).  I rejected publisher PDFs and full PDG PDF caching; all local
reference files are minimal text extracts with URLs, access date, and sha256.

For code coverage I reran targeted and broad greps over `quarkConstraints/`,
`qcd/`, `flavorConstraints/`, `neutrinos/`, `yukawa/`, `warpConfig/`,
`solvers/`, `scanParams/`, and `tests/`.  No tau-to-three-muons or four-lepton
LFV implementation was found.  The only adjacent LFV surface is the existing
`mu->e gamma` dipole checker in `flavorConstraints/muToEGamma.py` and its scan
call in `scanParams/scan.py`.  I therefore marked coverage `NO` and
implementation difficulty `MEDIUM`: a first implementation needs a lepton
contact-operator observable and matching convention, but no lattice or
hadronic long-distance machinery.

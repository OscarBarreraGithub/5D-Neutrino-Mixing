# PKA Worklog: T003

Process: `t -> c gamma`, top/Higgs/electroweak family.  I first read the
catalog plan, orchestrator decisions, Opus signoff, process template, and
README, then checked that `T003` did not already exist.  Neighboring top-FCNC
entries `T001` and `T005` set the local sidecar style.

Source searches used: `PDG t to c gamma branching fraction 4.2e-5`, `pdgLive
top gamma q`, `ATLAS arXiv 2205.02537 tq gamma`, `CMS TOP-21-013 t c gamma`,
and `Aguilar Saavedra hep-ph/0409342 t c gamma`.  The prompt's
\(4.2\times10^{-5}\) value is the ATLAS left-handed full-Run-2 result at 95%
CL, not the latest charm-specific limit.  CMS-TOP-21-013, published as Phys.
Rev. D 109 (2024) 072004, quotes
\(\mathcal{B}(t\to c\gamma)<1.51\times10^{-5}\) at 95% CL with
\(138~\mathrm{fb}^{-1}\), so I used CMS as the headline `pdg_or_equivalent`
source and retained ATLAS as a comparison source.  The PDG live t-quark page
has a generic \(t\to\gamma q\), \(q=u,c\), row at
\(<9.5\times10^{-6}\), but it is not charm separated; I recorded that as
context rather than replacing the CMS value.

Rejected or avoided sources: publisher PDFs from Elsevier/Springer and slide
decks found in search results.  I stored only minimal text snapshots under
`flavor_catalog/references/T003/`; no PDFs were added.  I also avoided using
the 2026 deep-learning phenomenology paper as a primary constraint source
because the catalog needs measured experimental limits first.

Coverage grep was run over `quarkConstraints/`, `qcd/`, `flavorConstraints/`,
`neutrinos/`, `yukawa/`, `warpConfig/`, `solvers/`, `scanParams/`, and
`tests/`.  No top-photon FCNC implementation was found.  Existing nearby
coverage remains neutral-meson \(\Delta F=2\) and lepton-sector
\(\mu\to e\gamma\).  Open issue for WA/CA: decide whether the public catalog
headline should be the latest CMS limit or an ATLAS/PDG-only convention.

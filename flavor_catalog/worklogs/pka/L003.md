# PKA Worklog: L003 mu-e Conversion in Aluminum

I read the required plan, orchestrator decisions, Opus signoff, process template,
and catalog README before creating files.  The key process-row instruction was
that L003 is a charged-lepton entry for coherent mu-e conversion in aluminum,
with Mu2e/COMET projections and initial code coverage `NOT-SEEN`.

Searches focused on primary or equivalent sources: `Mu2e Technical Design
single event sensitivity aluminum`, `Mu2e Run I Sensitivity Projections
2210.11380`, `COMET Phase-I Technical Design Report 1812.09018`, `COMET
Phase-II single event sensitivity 2.6e-17`, `Mu2e-II PIP-II 2203.07569`, and
`SINDRUM II mu-e conversion gold 7e-13`.  I used arXiv metadata/abstracts for
Mu2e, COMET, Mu2e-II, and the 2008 RS-flavor baseline.  For SINDRUM II I used
the JuSER bibliographic record rather than the Springer PDF, preserving the
publisher-PDF rule while retaining the DOI, journal metadata, and extracted
limit.  I rejected ResearchGate PDFs, publisher PDFs, and conference-slide PDFs
as tracked sources.  Search results also showed secondary review articles, but
the sidecar keeps the values tied to experiment sources instead.

The main source judgment is that L003 has no direct published aluminum bound in
the snapshots collected here.  I therefore recorded the SINDRUM II gold result
as the current all-target benchmark and kept Mu2e/COMET values as aluminum
projections.  The TeX explicitly avoids treating the gold limit as an aluminum
limit.

For coverage I ran the three required catalog grep commands over
`quarkConstraints`, `qcd`, `flavorConstraints`, `neutrinos`, `yukawa`,
`warpConfig`, `solvers`, `scanParams`, and `tests`, plus a targeted Mu2e/COMET/
SINDRUM/R_mue query.  The targeted query returned no hits.  The only adjacent
implementation is the existing `mu->e gamma` dipole checker in

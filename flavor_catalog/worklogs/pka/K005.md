# PKA Worklog: K005

Assigned process: `K_L -> pi^0 nu nubar`, the neutral kaon golden mode.
The plan v1 kaon-sector table identifies this unambiguously as `K005`, family
`kaon`, with initial coverage `NOT-SEEN`. I used that process id and did not
edit any catalog index files.

I first checked the current PDG live table for `K_L^0 -> pi^0 nu nubar`
(`S013R40`). The relevant PDG table entry is in units of `10^{-8}` and gives
`<0.22` at `90%` CL, document ID `AHN 2025`, technique `KOTO`, which I
converted to the sidecar value `B(K_L^0 -> pi^0 nu nubar) < 2.2e-9` at
`90%` CL. I then checked the KOTO 2025 PRL/arXiv record (`2411.11237`), which
confirms the 2021-data analysis: expected background
`0.252 +- 0.055(stat) +0.052/-0.067(syst)`, single-event sensitivity
`(9.33 +- 0.06(stat) +- 0.84(syst))e-10`, no signal-region events, and the
same `2.2e-9` limit.

For SM comparison I recorded two theory benchmarks rather than forcing one
canonical number: Brod--Gorbahn--Stamou 2021 gives
`2.59(29)e-11`, while Buras--Venturini 2022 gives
`(2.94 +- 0.15)e-11` under its explicit no-new-physics assumptions in
`|epsilon_K|` and `S_psi K_S`. I also added the 2026 NA62 charged-mode
measurement, `B(K^+ -> pi^+ nu nubar) = (9.6 +1.9/-1.8)e-11`, because it is
relevant to the kaon golden-mode program and charged/neutral cross-checks.
HFLAV was not used for a numerical average; its scope is heavy flavor and tau
averages, not rare kaon decays.

Coverage grep was run over `quarkConstraints/`, `qcd/`, `flavorConstraints/`,
`neutrinos/`, `yukawa/`, `warpConfig/`, `solvers/`, `scanParams/`, and
`tests/`, excluding notebooks. I found kaon `Delta F=2` mixing code and tests,
including `Delta_m_K`, but no KOTO/NA62 or `s->d nu nubar` rare-decay path.
I classified repo coverage as `NO` and implementation difficulty as `HIGH`.

# PKA Worklog: L010 tau -> 3e

I followed the required planning docs, orchestrator decisions, Opus signoff,
process template, and catalog README before creating new L010 files.  I first
checked that no `L010.tex`, `L010.yaml`, reference directory manifest, or PKA
worklog existed.  Search queries and source checks included "PDG tau e e e
branching fraction", "PDG live S035R58", "Belle tau decays into three leptons
2010 arXiv 1001.3221", "BaBar tau three charged leptons 2010 arXiv
1002.4550", "Belle II Physics Book tau LFV three leptons", and "LHCb tau
three electron decay".

The canonical value is the PDG live S035R58 tau datablock limit
`BR(tau^- -> e^- e+ e^-) < 2.7e-8` at 90% CL.  PDG selects the Belle 2010
Hayasaka et al. result based on `782 fb^-1`; the Belle arXiv source gives the
same value and the event-level inputs.  I recorded the BaBar 2010 result,
`<2.9e-8` at 90% CL with `468 fb^-1`, as the main independent cross-check.
I did not cache PDG or publisher PDFs.  All tracked sources are small text
snapshots with URLs, access dates, and sha256 hashes.

LHCb was checked because the prompt names Belle/BaBar/LHCb/PDG and L010 is the
companion to L009.  The LHCb source I found is for `tau -> 3mu`, not
`tau -> 3e`; it is therefore included only as context and contributes no L010
numerical value.  I also recorded the Belle II Physics Book for post-2008
prospects and the Agashe-Blechman-Petriello plus CFW sources for RS/lepton-LFV
context.

For code coverage I reran the required greps over `quarkConstraints/`, `qcd/`,
`flavorConstraints/`, `neutrinos/`, `yukawa/`, `warpConfig/`, `solvers/`,
`scanParams/`, and `tests/`.  No tau-to-three-electrons or four-lepton contact
LFV implementation was found.  The only adjacent charged-lepton LFV code is
the existing `mu->e gamma` dipole checker and its scan call, so I marked code
coverage `NO` and implementation difficulty `MEDIUM`.
